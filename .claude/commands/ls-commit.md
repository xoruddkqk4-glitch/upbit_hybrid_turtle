# ls-commit: 커밋 도우미 커맨드

변경된 파일을 확인하고, 쉬운 말로 요약한 뒤, 커밋 메시지를 제안하고 사용자 승인을 받아 커밋한다.
코딩을 모르는 사용자도 무슨 내용이 저장되는지 이해할 수 있도록 한국어로 설명한다.

---

## 진행 순서

### 1단계 — 변경 사항 확인

아래 두 가지를 확인한다:
- `git status` — 어떤 파일이 새로 생기거나 바뀌었는지
- `git diff HEAD` — 파일 안에서 실제로 무엇이 달라졌는지

커밋할 내용이 없으면 (변경 파일 없음):
```
현재 저장할 변경 사항이 없습니다.
파일을 수정한 뒤 다시 실행해 주세요.
```
라고 안내하고 종료한다.

---

### 2단계 — 변경 내용 요약 보고

확인한 내용을 아래 형식으로 보고한다:

```
📋 변경된 파일 목록

[새로 생긴 파일]
  + 파일명.py — 한 줄 설명

[수정된 파일]
  ✏️ 파일명.py — 무엇이 바뀌었는지 한 줄 설명

[삭제된 파일]
  - 파일명.py — 한 줄 설명
```

설명은 반드시 **쉬운 한국어**로 작성한다.
예: "나스닥 우상향 판단 로직 추가", "예수금 조회 방식 변경", "오류 처리 코드 보완"

---

### 3단계 — 커밋 메시지 제안

변경 내용을 바탕으로 커밋 메시지를 제안한다.

**메시지 형식:**
```
타입: 한 줄 요약 (한국어)
```

**타입 종류:**
| 타입 | 사용 상황 |
|------|---------|
| `feat` | 새로운 기능을 추가했을 때 |
| `fix` | 버그나 오류를 고쳤을 때 |
| `refactor` | 기능은 그대로인데 코드를 정리했을 때 |
| `docs` | 설명 문서(CLAUDE.md, 계획서 등)를 수정했을 때 |
| `test` | 테스트 코드를 추가하거나 수정했을 때 |
| `chore` | 설정 파일, 패키지 등 기타 변경 |

**제안 예시:**
```
제안하는 커밋 메시지:

  feat: 나스닥 우상향 필터 추가 (KODEX ETF 379800 기준 MA50/MA200 비교)

이 메시지로 저장할까요?
다른 메시지를 원하시면 직접 입력해 주세요. (그냥 '네'라고 하면 위 메시지로 저장)
```

사용자가 '네' 또는 별도 입력 없이 진행하면 제안 메시지를 사용한다.
사용자가 다른 메시지를 입력하면 그 메시지를 사용한다.

---

### 4단계 — 커밋 실행

승인된 메시지로 커밋을 실행한다:

1. 변경된 파일 전체를 스테이징 (`git add`)  
   단, 아래 파일은 절대 포함하지 않는다:
   - `.env` (API 키가 들어 있어 외부에 노출되면 안 됨)
   - `service_account.json` (Google 서비스 계정 키)
   - `held_coin_record.json`, `unheld_coin_record.json`, `trade_ledger.json`,
     `daily_snapshot.json`, `mock_balances.json` (런타임 상태 파일, 커밋 불필요)
   - `__pycache__/`, `*.pyc` (파이썬 자동 생성 파일)

2. 커밋 메시지에 아래를 자동으로 덧붙인다:
   ```
   Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
   ```

3. 커밋 완료 후 결과를 보고한다:
   ```
   ✅ 커밋 완료!

   커밋 ID: [앞 7자리]
   메시지: [커밋 메시지]
   저장된 파일: [파일 목록]
   ```

---

## 절대 규칙

1. **사용자 승인 없이 커밋하지 않는다** — 반드시 메시지를 먼저 보여주고 확인을 받는다
2. **`.env` 파일은 절대 커밋에 포함하지 않는다** — API 키가 들어 있어 보안 위험
3. **커밋 전에 변경 내용을 반드시 쉬운 말로 설명한다** — 사용자가 무엇이 저장되는지 알아야 한다
4. **git이 설정되지 않은 폴더에서는 안내 메시지를 출력하고 종료한다**

---

## git이 없는 경우 안내

현재 폴더가 git 저장소가 아니라면:
```
이 폴더는 아직 git이 설정되지 않았습니다.
git을 시작하려면 터미널에 아래 명령어를 입력하세요:

  git init
  git add .
  git commit -m "첫 번째 저장"
```
라고 안내하고 종료한다.

---

## 사용 예시

```
사용자: /ls-commit
```

Claude가 하는 일:
1. `git status`, `git diff HEAD` 로 변경 사항 확인
2. 변경된 파일과 내용을 한국어로 요약해서 보여줌
3. 커밋 메시지 제안
4. 사용자 승인 대기
5. 승인 후 커밋 실행 및 결과 보고

```
사용자: /ls-commit 오늘 작업 저장해줘
```

인자(추가 설명)가 있으면 커밋 메시지에 해당 내용을 반영해서 제안한다.
