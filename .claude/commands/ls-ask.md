# ls-ask: 질문 답변 전용 커맨드

사용자가 이 프로젝트의 코드·구조·동작 방식에 대해 궁금한 것을 물어볼 때 사용한다.
코딩을 모르는 사용자도 이해할 수 있도록 쉬운 말로 답변한다.

---

## 절대 규칙

1. **코드를 절대 수정하지 않는다** — 어떤 이유에서도 파일 내용을 바꾸지 않는다.
2. **답변만 한다** — 개선 제안, 리팩터링, 버그 수정 등 코드 변경으로 이어지는 행동을 하지 않는다.
3. **파일을 읽는 것은 허용** — 질문에 답하기 위해 코드를 읽고 분석하는 것은 가능하다.
4. **설명은 항상 쉬운 말로** — 기술 용어가 필요하면 바로 옆에 쉬운 설명을 덧붙인다.
5. **추측하지 않는다** — 확실하지 않으면 직접 파일을 읽고 확인한 뒤 답한다.

---

## 진행 방식

### 1단계 — 질문 파악

사용자의 질문이 무엇인지 정확히 파악한다.

- 특정 파일이나 함수에 관한 질문인지
- 전체 흐름·구조에 관한 질문인지
- 동작 방식(언제·왜·어떻게)에 관한 질문인지

### 2단계 — 필요한 파일 읽기

질문에 답하기 위해 필요한 파일을 읽고 내용을 파악한다.

- 관련 파일을 직접 열어 확인한다
- 필요하면 여러 파일을 함께 읽어 흐름을 추적한다

### 3단계 — 답변

아래 형식으로 답한다:

```
[질문을 한 줄로 다시 정리]

[쉬운 말로 핵심 답변 — 2~5줄]

[필요하면 코드의 해당 부분을 인용해서 설명]
예: `trade_ledger.py` 47번 줄에서 이렇게 처리합니다:
    ...코드 인용...
    → 이 코드가 하는 일: ...

[추가로 알면 좋은 것이 있으면 한 줄 보충]
```

---

## 답변하지 않는 것

- 코드를 고쳐드리겠다는 제안
- "이렇게 바꾸면 더 좋다"는 개선 의견
- 버그나 문제점 지적 후 수정 행동

궁금한 점에 대한 **설명·해석·분석** 만 제공한다.

---

## 사용 예시

```
사용자: /ls-ask ATR이 뭐야?
Claude: ATR은 'Average True Range'의 줄임말로, 코인 가격이 하루에 얼마나 많이 오르내리는지를
        숫자로 나타낸 것입니다. 이 프로젝트에서는 indicator_calc.py 파일에서 최근 20일치
        가격 변동을 평균 내어 계산합니다. 이 숫자가 클수록 가격이 많이 흔들리는 코인이고,
        작을수록 안정적인 코인입니다. 주문 수량을 결정할 때 이 값을 기준으로 삼습니다.

사용자: /ls-ask run_all.py 실행 순서가 어떻게 돼?
Claude: run_all.py를 실행하면 아래 순서로 동작합니다: ...
```
