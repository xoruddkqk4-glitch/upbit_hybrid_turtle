---

## 🎯 목표

급격한 변동이 자주 발생하는 특정 시간대에 2N 하드 손절을 일시적으로 보류한다.
이 시간대의 급락은 일시적인 경우가 많아서 가격이 회복되는 경우가 흔하다.
무조건 즉시 매도하는 대신 시간이 지나면 다시 정상적으로 감시를 재개한다.

**암호화폐 환경 적응:**
- 원본 LS 주식판은 "장 시작 직후(09:00~09:30)" 와 "장 마감 직전(15:00~15:30)" 를 블랙아웃으로 설정했다.
- Upbit 는 **24시간 거래** 이므로 해당 시간 개념이 성립하지 않는다.
- 대안 후보(선택적으로 도입):
  - **22:30~23:00 KST** — 미국 증시 개장 전후(암호화폐도 동조화되어 급변하는 구간)
  - **05:00~05:30 KST** — 미국 증시 마감 직후
  - **UTC 00:00 전후 30분** — 일봉 전환 시 자동 매매 봇들의 집중 청산 시각
- 현재 `upbit_hybrid_turtle/risk_guardian.py` 구현 버전은 **블랙아웃 기능을 제거**하고 24시간 동일하게 감시한다.
- 본 계획서는 "필요 시 재도입" 을 위한 참고 문서로 보류 상태 유지.

## PHASE 1 — 자료 수집

### P1-1 자료 수집 ✅

#### 수집 결과

**변경 대상 파일:** `risk_guardian.py` 단 하나

**현재 2N 손절 흐름 (run_guardian 함수 기준):**
```
run_guardian()
  └── 보유 코인 순회
        └── check_hard_stop(ticker, current_price, pos)
              ↓ True 이면
        place_exit_order(ticker, volume, "2N 하드 손절")
```

**check_hard_stop() 현재 코드 (Upbit 이식본):**
- `pos["stop_loss_price"]`(손절가) 가 없으면 False 반환
- `current_price <= stop_loss_price` 이면 True 반환 → 즉시 매도
- **LS 주식판에 있던 `_is_stop_blackout()` 호출은 이식 시점에 제거됨**

**시간 필터가 (재)들어갈 위치:**
- `check_hard_stop()` 내부에서 현재 시각을 확인하는 구간 추가
- 사용자가 정의한 고변동성 시간대면 → 손절 발동 보류, False 반환

## PHASE 2 — 설계

### P2-1 설계안 작성 ⬜

#### 설계 내용

**새로 만들 함수: `_is_stop_blackout()` (risk_guardian.py 내부 헬퍼)**

```
역할: 지금이 손절 보류 시간대인지 확인한다
입력: 없음 (함수 내부에서 현재 KST 시각을 읽음)
출력: True(보류 시간) / False(정상 감시 시간)

암호화폐용 보류 시간대 (후보 — 사용자 선택):
  - 22:30 ~ 23:00 KST  (미국 증시 개장 전후 고변동성 구간)
  - 05:00 ~ 05:30 KST  (미국 증시 마감 직후)
  - 08:45 ~ 09:15 KST  (UTC 00:00 일봉 전환 경계 전후)
  - 공백 (= 블랙아웃 비활성화, 24시간 감시)   ← 현재 Upbit 이식본 기본값
```

**check_hard_stop() 변경 내용:**
```
현재:
  현재가 ≤ 손절가 → True (즉시 매도)

재도입 시:
  현재가 ≤ 손절가 이면:
    _is_stop_blackout() 확인
      True(보류 시간) → 로그 출력 + False 반환 (이번 회차 스킵)
      False(정상 시간) → True 반환 (정상 매도)
```

**텔레그램 알림 추가:**
- 보류 시간에 손절 조건이 발동되면 텔레그램으로 아래 알림 발송
  ```
  ⚠️ 2N 손절 조건 발동 (보류 중)
  코인: 비트코인(KRW-BTC)
  현재가: 50,500,000원 ≤ 손절가: 51,000,000원
  사유: 고변동성 시간대(22:30~23:00) — 다음 회차에 재확인
  ```
