# 📋 계획서: 매수 후 최고가 - 2N 트레일링 손절

> 작성일: 2026-05-30
> 마지막 업데이트: 2026-05-30
> 현재 단계: P1 자료 수집 완료

---

## 🎯 목표

현재 2N 손절가는 "매수가 - 2N"으로 고정되어, 이후 가격이 아무리 올라가도 손절 기준이 함께 올라가지 않는다.
이를 "매수 후 최고가 - 2N" 방식으로 바꿔, 가격이 올라갈수록 손절 기준도 함께 올라가는 **트레일링(trailing) 손절**로 교체한다.

- 가격이 오르지 않으면 → 기존과 동일하게 동작 (매수가 = 최고가)
- 가격이 오른 뒤 급락하면 → 올라간 최고가 기준으로 더 일찍 손절 → 수익 보호

### P1-2 자료 검증 ✅

#### 검증 결과

- **ATR 이미 사용 가능**: `run_guardian()` 루프 안 241번 줄에서 `indicator_calc.get_all_indicators(ticker)` 호출 → `indicators["atr"]` 으로 바로 쓸 수 있음 ✅
- **`check_hard_stop()` 사용 필드**: `pos.get("stop_loss_price")` 단 하나. 이 값을 런타임에 갱신하면 기존 함수 수정 없이 트레일링 스탑으로 동작 ✅
- **피라미딩 시 `stop_loss_price` 갱신**: `place_pyramid_order()` 가 `new_avg_price - 2N` 으로 덮어씀. 이후 `run_guardian()` 이 다시 `current_price - 2N` 으로 덮어쓰므로 충돌 없음 ✅
- **`place_exit_order()` 의 독립 저장**: 자체적으로 `load → pop → save` 하므로, 루프 중간에 다른 저장이 있어도 안전하게 처리됨 ✅

**데이터 흐름 요약**

```
[매수 체결] turtle_order_logic
  → high_price_since_entry = 매수 체결가  (최초 저장)

[5분~10분마다] risk_guardian.run_guardian()
  → new_high = max(저장된 최고가, 현재가)
  → stop_loss_price = new_high - 2N  (덮어쓰기)
  → 현재가 ≤ stop_loss_price  →  전량 매도
  → 아니면 나머지 트레일링 스탑 체크
  → 루프 끝에 position_state 파일 저장
```
