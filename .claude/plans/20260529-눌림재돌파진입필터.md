## 🎯 목표

현재 터틀 전략은 신고가를 돌파한 뒤 **30분 이상 유지**되면 매수한다.
이 방식은 펌핑(급등 후 빠른 하락)에 속을 수 있다.

이 계획은 30분 가드를 제거하고, **"한 번 올랐다가 눌렸다가 다시 힘을 내며 최고점을 돌파할 때"** 매수하는
눌림→재돌파 필터로 교체한다. 이렇게 하면 진짜 추세가 확인된 뒤에 진입하게 되어
가짜 돌파 손실을 줄일 수 있다.

### P2-1 설계안 작성 ✅

#### 새 매수 전략 상태 흐름

```
[INACTIVE] ─── S1/S2 신호 True ──→ [WATCHING]
                                     현재가 > 이전 최고값 : 최고값 갱신
                                     현재가 < 이전 최고값 : 최고값 잠금 → [PULLBACK]
```

### P3-A target_manager.py 수정 ✅

#### 작업 목록

**1. `_update_since()` 함수 제거**
- 30분 가드 타임스탬프 관리 함수. 이 전략에서는 사용하지 않는다.

**2. `_update_peak_state()` 함수 신규 작성**

아래 로직을 구현한다:
```python
def _update_peak_state(
    prev_signal: bool, new_signal: bool,
    prev_peak: float | None, prev_locked: bool,
    current_price: float
) -> tuple:
    """peak 상태(peak_price, peak_locked, entry_ready)를 계산해서 반환한다.

    반환: (new_peak_price, new_peak_locked, entry_ready)
    """
    # 신호가 꺼지면 전체 초기화
    if not new_signal:
        return None, False, False

    # 신호가 방금 켜진 경우 (False → True) : peak 초기화
    if not prev_signal or prev_peak is None:
        return current_price, False, False

    # WATCHING 상태 (잠금 전)
    if not prev_locked:
        if current_price >= prev_peak:
            # 아직 상승 중 → 최고값 갱신
            return current_price, False, False
        else:
            # 최초 눌림 → 최고값 잠금
            return prev_peak, True, False

    # PULLBACK 상태 (잠금 후)
    if current_price > prev_peak:
        # 눌림 후 최고값 재돌파 → 진입 신호!
        return prev_peak, True, True
    else:
        # 아직 최고값 아래 → 대기
        return prev_peak, True, False
```

**3. 메인 루프에서 기존 `_since` 관련 코드 제거, peak 상태 적용**

기존:

**2. `check_pullback_rebreak()` 함수 신규 작성**

```python
def check_pullback_rebreak(ticker: str, signal_key: str, data: dict) -> bool:
    """눌림→재돌파 조건이 충족됐는지 확인한다.

    target_manager 가 계산해서 저장한 entry_ready 플래그를 읽는다.

    Args:
        ticker:     코인 티커 (예: KRW-BTC)
        signal_key: "s1" 또는 "s2"
        data:       unheld_coin_record 에서 읽은 코인 데이터

    Returns:
        True면 진입 신호 발생.
    """
    entry_ready = data.get(f"turtle_{signal_key}_entry_ready", False)
    peak_price  = data.get(f"turtle_{signal_key}_peak_price")

    if entry_ready and peak_price:
        name = get_watchlist().get(ticker, {}).get("name", ticker)
        print(
            f"[timer_agent] {name}({ticker}) ✅ 눌림→재돌파 진입 신호! "
            f"(최고값 {peak_price:,.0f}원 재돌파)"
        )
        return True

    # 상태별 대기 메시지 출력 (디버깅용)
    signal  = data.get(f"turtle_{signal_key}_signal", False)
    locked  = data.get(f"turtle_{signal_key}_peak_locked", False)
    name    = get_watchlist().get(ticker, {}).get("name", ticker)
    label   = signal_key.upper()

    if signal and not locked:
        print(
            f"[timer_agent] {name}({ticker}) ⏳ {label} WATCHING "
            f"(최고값 추적 중: {peak_price:,.0f}원)"
            if peak_price else
            f"[timer_agent] {name}({ticker}) ⏳ {label} WATCHING (최고값 미확인)"
        )
    elif signal and locked:
        print(
            f"[timer_agent] {name}({ticker}) ⏳ {label} PULLBACK "
            f"(최고값 {peak_price:,.0f}원 돌파 대기 중)"
            if peak_price else
            f"[timer_agent] {name}({ticker}) ⏳ {label} PULLBACK (최고값 미확인)"
        )

    return False
```

- **기존 JSON 호환**: 코드가 배포된 직후 실행 시 기존 `unheld_coin_record.json`에는
  `turtle_s1_peak_price` 필드가 없다. `.get()` 기본값을 쓰기 때문에 자동으로 초기화되며,
  다음 실행 사이클부터 정상 동작한다. 별도 마이그레이션 불필요.

- **`_since` 필드**: 기존 JSON에 남아 있을 수 있지만 새 코드에서는 읽지도 쓰지도 않으므로
  시간이 지나면 자연스럽게 사라진다.

- **10시 필터 미적용**: 한국 주식 전략에서 논의된 10시 필터는 업비트(24시간 운영)에
  해당하지 않으며, 현재 코드에도 없다. 별도 처리 불필요.

- **진입 경로 레이블**: `TURTLE_S1` / `TURTLE_S2` 는 유지. Google Sheets 원장과의 호환 깨지지 않음.
