---

## 🎯 목표

사용자가 업비트 앱이나 웹에서 직접 손으로 사고팔았을 때에도, 그 매수·매도 기록을
Google Sheets 체결원장(시트1)에 자동으로 남기도록 만든다.
지금은 잔고 비교로 "수동 거래가 있었다는 사실"만 감지하고 시트에는 기록이 안 남는데,
이 격차를 메운다.

핵심 아이디어: **API 호출을 아끼기 위해 매번 모든 종목의 체결 내역을 조회하지 않는다.
잔고가 기록과 어긋난 종목에 대해서만 Upbit 의 주문 조회 API 를 호출해
"내가 안 낸 주문(=수동 거래)" 을 찾아 시트에 기록한다.**

## 🎯 최종 결과 요약

### 변경 파일
1. **`upbit_client.py`** — 새 공개 함수 2개 추가
   - `get_execution_detail(order_no)` — `_get_execution_from_order` 의 외부 공개 래퍼
   - `fetch_recent_done_orders(ticker)` — 종목별 done 주문 목록 조회

2. **`trade_ledger.py`** — 3가지 변경
   - `VALID_SOURCES` 에 `MANUAL_BUY`, `MANUAL_SELL` 추가
   - 한글 라벨 매핑에 "수동 매수", "수동 매도(익절/손절)" 추가
   - `get_recorded_order_nos()` helper 추가 — 중복 기록 방지

3. **`balance_sync.py`** — 핵심 로직
   - `import trade_ledger` 추가
   - `_record_manual_trades(ticker, coin_name, ref_avg_price)` — 수동 거래 시트 기록 helper
   - `_apply_manual_buys_to_record(entry, manual_buys, atr_n)` — 수동 매수 반영 시 평균가·손절가·피라미딩가 재계산
   - 잔고 불일치 ①②③ 분기 안에 위 helper 호출 추가

### 작동 흐름
- 잔고가 기록과 일치 → 추가 Upbit API 호출 0회 (기존과 동일)
- 잔고가 불일치 → 그 종목 1회만 `get_order(state="done")` + 각 신규 주문당 `get_order(uuid)` 1회 추가
- ledger 의 `order_no` 집합과 대조하여 "자동매매가 안 낸 주문 = 수동 거래" 자동 식별
- `trade_ledger.append_trade()` 단일 진입점으로 시트1 기록 + 텔레그램 알림 일관 처리
- 수동 매수가 있었다면 `held_coin_record.json` 의 평균가·손절가·다음 피라미딩가도 함께 재계산

### 사용자 결정 사항 (반영 완료)
- 수동 매도 손익 계산 기준: **내 기록(`held_coin_record.json`)의 `avg_buy_price`**
- 수동 부분 매수 시 자동매매 기록 갱신: **평균가까지 같이 갱신**

### 알아둘 점 (사각지대)
- 한 사이클(5분) 안에서 수동 매수→매도가 완결되어 잔고에 흔적이 안 남으면 감지 불가
- Upbit `get_order(state="done")` 는 한 번에 최근 100건 한도
