### P1-2 기존 코드 LOVELY_COIN_LIST 사용 위치 파악 ✅

> 목적: 어떤 파일들이 LOVELY_COIN_LIST 를 직접 참조하는지 파악해서 수정 범위를 확정한다.

#### 확인 결과

| 파일 | 참조 수 | 주요 용도 |
|------|--------|---------|
| `turtle_order_logic.py` | 15곳 | `if ticker not in LOVELY_COIN_LIST` 안전장치, 코인명 조회, 진입/피라미딩 목록 반복 |
| `timer_agent.py` | 7곳 | 안전장치 체크, 코인명 조회 |
| `risk_guardian.py` | 7곳 | 안전장치 체크, 코인명 조회, 감시 대상 반복 |
| `target_manager.py` | 4곳 | `unheld_tickers = [t for t in LOVELY_COIN_LIST ...]` 미보유 목록 생성, 코인명 조회 |
| `config.py` | 2곳 | 상수 정의, `get_coin_name()` 함수 |

**핵심 패턴 3가지 (교체 방법):**
```
패턴①: if ticker not in LOVELY_COIN_LIST  →  if ticker not in get_watchlist()
패턴②: for ticker in LOVELY_COIN_LIST     →  for ticker in get_watchlist()
패턴③: LOVELY_COIN_LIST.get(ticker, {}).get("name", ticker)
        →  get_watchlist().get(ticker, {}).get("name", ticker)
```

**섹터/테마 캐시 고려:**
- Upbit 는 섹터 API 가 없어서 본 프로젝트에서 테마 캐시 모듈 자체가 존재하지 않음
- LS 주식판의 `sector_cache.py` 에 해당하는 수정 작업 불필요

```

함수 목록 (예정):
  run_premarket_screening()
    → 08:40 배치. 후보 50~80개 선별 → coin_candidates.json 저장
    내부 흐름:
      1. _fetch_all_krw_tickers()  — pyupbit.get_tickers(fiat="KRW")
      2. _fetch_ticker_snapshot()  — /v1/ticker 로 전체 24h 거래대금·고저가 수집
      3. _apply_value_filter()     — 24h 거래대금 100억 이상만 통과
      4. _fetch_24h_high_pool()    — 24h 고가 근접도(price / 24h_high) 0.95 이상 풀 (보조)
      5. _merge_pools()            — 합집합, 출처 태그 부여 (volume/highprice/both)
      6. _calc_indicators()        — 남은 코인만 ATR·MA 계산 (일봉 60개)

      8. _save_candidates()        — coin_candidates.json 저장

  run_market_open_screening()
    → 09:05 배치. 후보를 당일 거래대금으로 재정렬 → dynamic_watchlist.json 저장
    내부 흐름:
      1. _load_candidates()        — coin_candidates.json 로드
      2. _refresh_volume()         — /v1/ticker 재호출 → 당일 거래대금 갱신
      3. _adjust_scores()          — 전일 후보 순위 조정, 신규 진입 코인 소량 보완
      4. _apply_whitelist()        — watchlist_config.json 화이트리스트 합집합

      7. _save_watchlist()         — dynamic_watchlist.json 저장 + 텔레그램 알림
```

#### 상세 설계: coin_screener.py 전체

```
# 설정 상수
VOLUME_POOL_SIZE  = 80            # 거래대금 상위 수집 수
CANDIDATE_TARGET  = 60            # 후보 목표 수 (50~80 범위)
WATCHLIST_SIZE    = 30            # 최종 감시 코인 수

VALUE_MIN         = 10_000_000_000  # 24h 거래대금 하한 (100억 원)
ATR_RATIO_MIN     = 0.020           # ATR/종가 최소 (2.0%)
OVERHEATING_RATIO = 0.40            # 20일선 대비 이격도 상한 (40%)
HIGH_NEAR_RATIO   = 0.95            # 24h 고가 근접도 풀 포함 기준 (price/high ≥ 0.95)

SCORE_W_VOL  = 0.4   # 거래대금 순위 가중치
SCORE_W_HIGH = 0.4   # 고가 근접도 가중치
SCORE_W_MA   = 0.2   # 정배열 가중치

CANDIDATES_PATH = "coin_candidates.json"
WATCHLIST_PATH  = "dynamic_watchlist.json"
CONFIG_PATH     = "watchlist_config.json"
```

```
[08:40 배치] run_premarket_screening()
흐름 1. _fetch_all_krw_tickers()
         → pyupbit.get_tickers(fiat="KRW") 전체 KRW 마켓 수집 (200개 내외)

     2. _fetch_ticker_snapshot()
         → 100개씩 배치로 /v1/ticker 호출 → 24h 거래대금·고저가 수집

     3. _apply_value_filter()
         → acc_trade_price_24h < 100억 → 탈락
         → 거래대금 내림차순 상위 80개만 남김

     4. _fetch_24h_high_pool()  (보조 풀)
         → trade_price / high_price ≥ 0.95 인 코인 추가

     5. _merge_pools()
         → 두 풀 합집합, 중복 제거
         → 각 코인에 "출처" 태그 부여 (volume / highprice / both)

     6. _calc_indicators()
         → 남은 코인(80개 이하)에 대해서만
         → indicator_calc.get_screener_indicators(ticker) 호출
         → ATR/종가 < 2.0% 코인 탈락
         → 20일선 대비 이격도 > 40% 코인 탈락

     7. _score_and_rank()
         → 거래대금 순위 점수 = 1 - (volume_rank / 후보수)
         → 고가 근접도 점수   = price / high_60d
         → 정배열 점수        = 1 if ma5 > ma20 else 0
         → 종합 점수 = 0.4×거래대금순위점수 + 0.4×고가근접도 + 0.2×정배열
         → 내림차순 정렬, 상위 CANDIDATE_TARGET(60)개 선발

     8. _save_candidates()
         → coin_candidates.json 저장
         → 텔레그램: "08:40 후보 N개 선정 완료"

[09:05 배치] run_market_open_screening()
흐름 1. _load_candidates()
         → coin_candidates.json 로드
         → 없으면 "후보 파일 없음" 경고 후 오늘 run_premarket_screening() 재실행

     2. _refresh_volume()
         → /v1/ticker 재호출 → 당일 누적 거래대금으로 재정렬
         → 후보 코인의 value 와 volume_rank 를 당일 값으로 갱신
         → 당일 신규 진입 코인(전일 후보 아님): 소량 지표 계산 후 후보 추가

     3. _rescore()
         → 갱신된 volume_rank 로 스코어 재계산

     4. _apply_whitelist_blacklist()
         → watchlist_config.json 로드 (없으면 건너뜀)
         → blacklist 티커 제거
         → whitelist 티커 Top 30에 강제 포함

     5. _finalize_top30()
         → 스코어 내림차순 상위 30개 확정 (화이트리스트 포함 후 채움)

     6. _save_watchlist()
         → dynamic_watchlist.json 저장
         → 텔레그램: "최종 감시 코인 30개 확정" + 상위 10개 목록
```
