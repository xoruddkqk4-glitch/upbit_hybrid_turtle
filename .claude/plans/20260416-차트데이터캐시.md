## 🎯 목표

`run_all.py` 가 실행될 때마다 코인별로 반복하는 일봉·240분봉 API 호출을 없애서
프로그램 1회 실행 시간을 줄인다. Upbit 는 public API 에 **초당 10회 제한**이 있어,
30개 코인 × 2개 차트 = 60회 호출을 매번 반복하면 대기 시간이 무시할 수 없다.

- **일봉(하루 캔들)**: 09:05 코인 확정 시점에 30개 전부 받아 `daily_chart_cache.json` 에 저장.
  이후 실행에서는 파일에서 꺼내 씀 (유효 기간: 당일 UTC 자정 경계까지).
- **240분봉(4시간 캔들)**: 같은 시점에 함께 저장.
  유효 시간 30분 — 30분이 지난 항목만 재조회해서 덮어쓰고, 아직 신선한 항목은 파일 그대로 사용.

## PHASE 2 — 설계

### P2-1 설계안 작성 ✅

> 목적: 어떤 구조로 캐시를 만들고, 어떤 함수를 고칠지 설계도를 그린다.

#### 신규 파일: `daily_chart_cache.py`

4개의 함수로 구성된다.

| 함수 | 역할 | 호출 시점 |
|------|------|---------|
| `build_cache(watchlist)` | 30개 전체 일봉 60·240분봉 API 조회 후 JSON 저장 | `coin_screener market_open` 완료 직후 (1회) |
| `get_daily_cached(ticker, count)` | 오늘 날짜의 일봉 캐시 반환. 없으면 빈 리스트 | `indicator_calc`, `target_manager` 에서 호출 |
| `get_minute240_cached(ticker)` | 240분봉 캐시 반환. 30분 초과면 빈 리스트 | `indicator_calc`, `target_manager` 에서 호출 |
| `update_minute240_cache(ticker, data)` | 240분봉 재조회 후 해당 코인 캐시만 덮어쓰기 | 캐시 만료 시 `indicator_calc` 내부에서 호출 |

#### 신규 JSON 파일: `daily_chart_cache.json`

```json
{
  "KRW-BTC": {
    "daily":                [...],  ← 일봉 60개 (ohlcv 리스트)
    "daily_date":           "2026-04-16",
    "minute240":            [...],  ← 240분봉 25개
    "minute240_fetched_at": "2026-04-16 09:05:30"
  },
  "KRW-ETH": { ... }
}
```

#### 변경 파일 및 변경 내용

**`coin_screener.py`** — `run_market_open_screening()` 내 `initialize_unheld_record()` 호출 뒤에 추가:
```python
# 일봉·240분봉 캐시 빌드 (이후 run_all.py 에서 API 재호출 없이 사용)
daily_chart_cache.build_cache(watchlist)
```

**`indicator_calc.py`** — `get_all_indicators()` 수정:
- `upbit_client.get_daily_chart()` → `daily_chart_cache.get_daily_cached()` 우선, 없으면 API 폴백
- `upbit_client.get_minute_chart()` → `daily_chart_cache.get_minute240_cached()` 우선, 없으면 API 재조회 + `update_minute240_cache()` 호출

**`target_manager.py`** — `run_update()` 수정:
- `upbit_client.get_daily_chart(ticker, count=60)` → 캐시 우선, 없으면 API 폴백
- `upbit_client.get_minute_chart(ticker, minute=240, count=25)` → 캐시 우선, 없으면 API 재조회 + 캐시 갱신
- **`time.sleep(0.2)` 처리**: 캐시에서 읽을 때는 API 를 안 부르므로 이 대기가 불필요해짐.
  → 두 가지 선택지 (사용자 결정 필요):
  - **A안**: 캐시 히트 시 sleep 스킵, 폴백 시에만 sleep (추가 시간 단축)
  - **B안**: 캐시 여부와 무관하게 sleep 유지 (보수적, rate limit 안전)

#### 데이터 흐름

```
09:05  coin_screener market_open
       ↓ _save_watchlist (30개 확정)
       ↓ target_manager.initialize_unheld_record()
       ↓ daily_chart_cache.build_cache(watchlist)
            → 30개 × (일봉60 + 240분봉25) API 조회 (0.2초 간격 × 30 ≈ 12초)
            → daily_chart_cache.json 저장

10:00+ run_all.py
  STEP 2 risk_guardian
    → indicator_calc.get_all_indicators(ticker)
         → get_daily_cached(ticker)   ← JSON 파일 읽기 (0.01초)
         → get_minute240_cached(ticker) ← TTL 확인 후 JSON 파일 읽기 (0.01초)
  STEP 3 target_manager
    → get_daily_cached(ticker)   ← JSON 파일 읽기
    → get_minute240_cached(ticker) ← TTL 확인
         (만료면) upbit_client.get_minute_chart() → update_minute240_cache()
  STEP 5 turtle_order_logic
    → indicator_calc.get_all_indicators(ticker) ← 캐시 재사용
```

### P3-3 `indicator_calc.py` — 캐시 우선 읽기로 변경 ⬜

**변경 대상:** `get_all_indicators()` 함수
**변경하지 않는 함수:** `get_screener_indicators()` (08:40 배치 = 캐시 빌드 전)

**변경 전 (현재):**
```python
daily = upbit_client.get_daily_chart(ticker, count=60)
...
minute_data = upbit_client.get_minute_chart(ticker, minute=240, count=25)
```

**변경 후:**
```python
# 일봉: 캐시 우선 → 없으면 API 직접 조회
import daily_chart_cache
daily = daily_chart_cache.get_daily_cached(ticker, count=60)
if not daily:
    print(f"[indicator] {ticker} 일봉 캐시 없음 → API 직접 조회")
    daily = upbit_client.get_daily_chart(ticker, count=60)

# 240분봉: 캐시 우선 → 만료/없으면 API 재조회 + 캐시 갱신
minute_data = daily_chart_cache.get_minute240_cached(ticker)
if not minute_data:
    print(f"[indicator] {ticker} 240분봉 캐시 없음/만료 → API 조회 후 캐시 갱신")
    minute_data = upbit_client.get_minute_chart(ticker, minute=240, count=25)
    if minute_data:
        daily_chart_cache.update_minute240_cache(ticker, minute_data)
```
