## 🎯 목표

Upbit Open API 를 이용해서 암호화폐(KRW 마켓의 BTC/ETH 등 7개 코인)를 자동으로 사고파는 프로그램을 만든다.
전설적인 터틀 트레이딩 방식으로 얼마나 살지 결정하고, 가짜 급등을 피하기 위해 30분 동안 가격이
유지되는지 확인한 뒤에만 매수하는 "하이브리드" 전략을 자동으로 실행한다.

> 이 계획서는 `ls_hybrid_turtle` (국내주식 LS증권 버전) 의 동일 문서를 Upbit 환경에 맞게 포팅한 것이다.
> 구조·단계·체크리스트는 원본과 동일하게 유지하고, API·식별자·시장 특성만 암호화폐용으로 교체했다.

**[필요한 API 호출 방법 확인]**

| 용도 | 함수 | 호출 방법 | 응답 |
|------|------|---------|------|
| 로그인 | - | `pyupbit.Upbit(access_key, secret_key)` | Upbit 객체 (이후 주문·잔고용) |
| 현재가 조회 (여러 개) | `get_current_price` | `pyupbit.get_current_price(["KRW-BTC", "KRW-ETH", ...])` | `{"KRW-BTC": 53100000, ...}` |
| 일봉 차트 (ATR, MA, 신저가) | `get_ohlcv` | `pyupbit.get_ohlcv("KRW-BTC", interval="day", count=60)` | pandas DataFrame (open/high/low/close/volume) |
| 분봉 차트 (240분봉 20MA) | `get_ohlcv` | `pyupbit.get_ohlcv("KRW-BTC", interval="minute240", count=25)` | pandas DataFrame |
| 시장가 매수 | `buy_market_order` | `upbit.buy_market_order("KRW-BTC", krw_amount)` | 주문 응답 dict (uuid 포함) |
| 시장가 매도 | `sell_market_order` | `upbit.sell_market_order("KRW-BTC", volume)` | 주문 응답 dict |
| 잔고 조회 | `get_balances` | `upbit.get_balances()` | `[{"currency": "BTC", "balance": "0.01", "avg_buy_price": "53000000", ...}, ...]` |
| 총 자산 KRW | `myUpbit.GetTotalRealMoney` | 래퍼 함수 — 보유 코인 평가금 + KRW 합산 | float (원) |
| 주문 상태 | `get_order` | `upbit.get_order(uuid)` | 체결 상세 |

**[중요 발견사항]**
- **시장가 매수는 "KRW 금액"을 넘긴다**: `buy_market_order("KRW-BTC", 10000)` → 1만원어치 매수 (수량 아님!)
- **시장가 매도는 "코인 수량"을 넘긴다**: `sell_market_order("KRW-BTC", 0.00015)` → 0.00015 BTC 매도
- **최소 주문금액**: KRW 마켓은 **5,000 원 미만 주문 불가**
- **소수점 수량**: 주식과 달리 코인은 소수 허용 (예: 0.00012345 BTC)
- Upbit 에는 `A + 종목코드` 같은 모의투자 전용 포맷이 없음. 모의투자는 자체적으로 `mock_balances.json` 에 기록하는 방식으로 처리.

**[환경변수 (.env) 항목]**
- `UPBIT_ACCESS_KEY` — Upbit Access Key
- `UPBIT_SECRET_KEY` — Upbit Secret Key
- `UPBIT_PAPER_TRADING` — 모의투자 여부 (True/False)
- `UPBIT_ACCOUNT_LABEL` — 계좌 별칭 (로그·원장용)
- `TELEGRAM_BOT_TOKEN` — 텔레그램 봇 토큰
- `TELEGRAM_CHAT_ID` — 텔레그램 채팅 ID
- `GOOGLE_SERVICE_ACCOUNT_JSON` — Google 서비스 계정 JSON 경로
- `GOOGLE_SPREADSHEET_TITLE` — Google Sheets 시트 제목
- `GOOGLE_DRIVE_FOLDER_ID` — (선택) Google Drive 폴더 ID

**[P1-1에서 놓쳤던 중요 발견]**

- **멀티 현재가 조회**: `pyupbit.get_current_price` 에 리스트를 넘기면 한 번의 API 호출로 전체 현재가를 받을 수 있어, LS 의 t8407 과 동일한 "한 번에 7개" 효과를 낸다.
- **수수료**: Upbit KRW 마켓 기본 수수료는 0.05% (쿠폰·멤버십에 따라 다름). 주문 응답의 `paid_fee` 필드로 체결 원장에 기록.

### ⑥ target_manager.py — 동적 목표가 관리

**`unheld_coin_record.json` 구조:**
```json
{
  "KRW-BTC": {
    "pending_target": 53100000,
    "above_target_since": "2026-04-21 10:30:00",
    "last_updated": "2026-04-21 10:00:00"
  }
}
```
> `above_target_since` 가 null 이면 아직 목표가 미달, 값이 있으면 그 시각부터 30분 카운트 시작

| 함수 | 하는 일 |
|------|---------|
| `load_unheld_record()` | unheld_coin_record.json 읽기 |
| `save_unheld_record(record)` | JSON 저장 |
| `calc_pending_target(ticker, current_price, indicators)` | `max(현재가×1.02, 240분봉20MA×1.005)` 계산 |
| `update_above_target_time(ticker, current_price, record)` | 현재가 ≥ pending_target 이면 타이머 시작/유지, 아니면 초기화 |
| `run_update()` | 전체 미보유 코인 목표가 갱신 (메인 실행 함수) |

| 함수 | 하는 일 |
|------|---------|
| `check_hard_stop(ticker, current_price, pos_state, indicators)` | 현재가 ≤ 손절가 이면 손절 신호 |
| `check_trailing_stop(ticker, current_price, indicators)` | 10일 신저가 경신 또는 5MA 하향 돌파 시 익절 신호 |
| `place_exit_order(ticker, volume, reason)` | 전량 매도 주문 + 상태 삭제 + 원장 기록 + 알림 |
| `run_guardian()` | 전체 보유 코인 손절/익절 감시 (메인 실행 함수) |

### ⑩ run_all.py — 통합 실행기 (배치)

- `get_total_capital()` — KRW + 보유 코인 평가금 합산
- `get_balance()` — 보유 코인 잔고 목록 (KRW 마켓 티커 기준)

- pyupbit 래퍼·지표·주문 유틸 (기존 파일 그대로 사용)

- `calc_ma(close_list, period)` — 이동평균
- `calc_10day_low(ohlcv_list)` — 10일 신저가
- `calc_n_day_high(ohlcv_list, n)` — N일 신고가 (터틀 신호용)
- `get_all_indicators(ticker, upbit_client)` — 한 번에 모두 계산

**trade_ledger.py** ← [보완 #2: KST 적용]
- `append_trade(record)` — JSON + Google Sheets 저장
- `_generate_record_id()` — 고유 ID 생성
- 모든 타임스탬프는 `pytz` KST 기준

**telegram_alert.py**
- `SendMessage(msg)` — 텔레그램 봇 발송 (1회 실패 시 1회 재시도 후 포기)

**target_manager.py** ← [보완 #2: KST 적용]
- `calc_pending_target(ticker, current_price, indicators)` — max(현재가×1.02, 240분봉20MA×1.005)
- `update_above_target_time(ticker, current_price, record)` — 목표가 이상이면 KST 타임스탬프 기록, 미달이면 null 초기화
- `run_update()` — 미보유 코인 전체 목표가 갱신

**timer_agent.py**
- `check_30min_passed(ticker, unheld_record)` — above_target_since 로부터 30분 경과 확인
- `run_timer_check()` — 전체 체크 후 통과 티커 리스트 반환

**turtle_order_logic.py** ← [보완 #3: 피라미딩 시 손절가 갱신 / 보완 #4: 티커 범위 체크]
- `calc_unit_size(ticker, price, atr_n, total_capital)` — Unit 수량 계산 (5,000 KRW 최소금액 필터)
- `place_entry_order(ticker, volume, price, entry_source)` — 1차 진입 + held_coin_record 저장
  - stop_loss_price = price - 2×atr_n
  - next_pyramid_price = price + 0.5×atr_n
- `place_pyramid_order(ticker, volume, price, atr_n)` — 피라미딩 + last_buy_price·stop_loss_price·next_pyramid_price 갱신
- `check_pyramid_trigger(ticker, current_price, pos_state, atr_n)` — 현재가 ≥ next_pyramid_price 확인
- `run_orders(entry_signals)` — 진입 신호 처리 + 피라미딩 체크
  - **모든 주문 전 `LOVELY_COIN_LIST` 포함 여부 확인**

**risk_guardian.py** ← [보완 #1: 수익권 조건 / 보완 #4: 티커 범위 체크]
- `check_hard_stop(ticker, current_price, pos_state, atr_n)` — 현재가 ≤ last_buy_price - 2×atr_n
- `check_trailing_stop(ticker, current_price, pos_state, indicators)`:
  - 10일 신저가 경신 → 무조건 청산
  - 5MA 하향 돌파 → **current_price > 평균매입단가(수익권)일 때만** 청산
- `place_exit_order(ticker, volume, reason)` — 전량 매도 + held_coin_record 삭제
  - **주문 전 `LOVELY_COIN_LIST` 포함 여부 확인**
- `run_guardian()` — 전체 보유 코인 감시

**run_all.py**
