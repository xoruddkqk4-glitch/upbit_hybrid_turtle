## 🎯 목표

지금은 `손익차트` 워크시트의 그래프가 **일(日) 단위 고정**으로 그려진다.
사용자가 **일 / 주 / 월 / 분기 / 년** 다섯 가지 중 하나를 골라 같은 데이터를
다른 시간 단위로 묶어 볼 수 있게 만든다.

핵심 아이디어:
- 원본 데이터(`포트폴리오 추이` 시트의 날짜별 실현손익)는 그대로 두고,
- **`build_series_from_portfolio()`** 에 "묶기 단위" 인자를 받아 일/주/월/분기/년
  단위로 합산한 시계열을 만들고, 그 결과를 `손익차트` 시트에 쓰면 된다.

#### 검증 결과

**1️⃣ 합산 규칙 — 당일 실현손익을 기간 합산해도 안전한가? → ✅ 안전**
- `trade_ledger.calc_realized_pnl_today()` (line 538~567) 은 그날의 모든 SELL `profit_amount` 합
- `_upsert_portfolio_direct` 가 매도 시마다 G열에 그 결과를 덮어씀 → **그날의 "마지막 행"이 곧 그날 전체 실현손익**
- `build_series_from_portfolio` 도 이미 같은 규칙(`daily_last[date_str]`)으로 마지막 값을 사용
- 따라서 주/월/분기/년 합산도 "각 날짜의 마지막 값을 합산" 으로 일관성 유지

**2️⃣ 누적 실현손익 규칙 → ✅ 마지막 값 사용이 정답**
- `누적수익금(원)` (J열) 은 그 시점의 누적 합 → 같은 날 마지막 행이 가장 최신
- 기간 단위로 묶을 때도 **그 기간 안의 마지막 날짜의 마지막 값** 이 그 기간의 최종 누적

**3️⃣ 주/월/분기/년 키 생성 (Python 기본 기능으로 가능)**
| 단위 | 키 형식 후보 | 생성 방법 |
|------|-------------|----------|
| 일 | `2026-05-29` | `dt.strftime("%Y-%m-%d")` |
| 주 | `2026-W22` (ISO) | `dt.strftime("%G-W%V")` (월요일 시작, ISO 표준) |
| 월 | `2026-05` | `dt.strftime("%Y-%m")` |
| 분기 | `2026-Q2` | `f"{dt.year}-Q{(dt.month-1)//3+1}"` (1~3=Q1, 4~6=Q2…) |
| 년 | `2026` | `dt.strftime("%Y")` |

→ 외부 라이브러리 불필요. 표준 `datetime` 모듈만으로 5가지 모두 가능.

**4️⃣ 데이터 적을 때 차트 → ✅ Google Sheets 정상 동작**
- 점이 1개라도 막대/선 차트 정상 표시 (구글 차트 기본 동작)
- 0원 행이 있어도 기준선 위 시계열로 자연 표시 (현재 코드 주석에도 명시: line 19~21)

**5️⃣ 자동매매 영향 → ✅ 0%**
- `pnl_chart.py` 는 `run_all.py` 어디서도 호출하지 않음 (`run_daily.py` 와 `trade_ledger.refresh_sheets_after_sell()` 만 호출)
- 차트 기능에 오류가 나도 `try/except` 로 감싸져 있어 매매 흐름 차단 안 됨

**6️⃣ 호환성 → ✅ 기존 호출자 수정 불필요**
- `run_pnl_chart()` 호출자 2곳은 모두 인자 없이 호출

#### 위 결정에서 파생되는 설계 방향
- `손익차트` 시트에 **드롭다운 셀(F1)** 설치 — 옵션 5개 (일/주/월/분기/년)
- 메인 차트(항상 표시): **일 단위** — 현재와 동일, 데이터는 A~C열 그대로
- 보조 차트(드롭다운 선택에 따라): **선택한 단위**로 묶은 데이터 — 데이터는 별도 영역(E~G열 등)
- 드롭다운에서 "일" 선택 시 → 보조 차트는 생성 생략 (메인이 이미 일 단위라 중복 방지)
- 데이터/차트 갱신 시점: 기존과 동일 (`run_pnl_chart()` 호출 시 = 매도 즉시 + 일 1회 23:55)
- 사용자가 드롭다운만 바꿔도, 다음 갱신 때 보조 차트가 새 단위로 다시 그려짐
