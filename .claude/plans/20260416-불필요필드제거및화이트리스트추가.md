## 배경 분석 (이미 완료)

이번 작업 전에 코드를 분석한 결과, 아래 사실이 확인됐다.

### 왜 제거하는가?

`pending_target` 과 `reference_price` 는 **"사기 전(미보유 코인)"** 에만 필요한 정보다.

| 파일 | pending_target 사용 여부 |
|------|------------------------|
| `risk_guardian.py` | ❌ 안 씀 — 손절가·10일신저가·5MA 만 사용 |
| `turtle_order_logic.py` | ❌ 안 씀 — 피라미딩은 next_pyramid_price 만 사용 |
| `timer_agent.py` | ❌ 안 씀 — unheld_coin_record 만 읽음 |
| `target_manager.py` | ❌ 갱신만 하고 읽어서 판단에 쓰지 않음 |

즉, 저장하고 갱신하는 코드는 있지만 **읽어서 실제 매매 판단에 쓰는 코드가 없다.**

### 보유 코인에 실제로 필요한 정보

| 정보 | 역할 |
|------|------|
| `stop_loss_price` | 하드 손절 기준가 |
| `avg_buy_price` | 평균 매입단가 (5MA 익절 조건 수익권 판단) |
| `next_pyramid_price` | 피라미딩 추가 매수 기준가 |
| `current_unit` | 현재 몇 번 샀는지 |
| `max_unit` | 최대 매수 횟수 |
| `last_buy_price` | 마지막 매수가 (손절가·피라미딩 기준가 재계산용) |
| `total_volume` | 현재 보유 수량 (평균가 재계산용) |

`pending_target`·`reference_price` 는 이 목록에 없다.

## PHASE 3 — 구현

### P3-1 turtle_order_logic.py — 불필요 필드 제거 ⬜

**변경 위치:** `place_entry_order()` 함수 내 포지션 상태 저장 부분

**현재 코드:**
```python
position_state[ticker] = {
    "current_unit":       1,
    "last_buy_price":     price,
    "avg_buy_price":      price,
    "stop_loss_price":    stop_loss_price,
    "next_pyramid_price": next_pyramid_price,
    "entry_type":         entry_type,
    "max_unit":           max_unit,
    "total_volume":       volume,
    "entry_source":       entry_source,
    "pending_target":     price * 1.02,   ← 제거 대상
    "reference_price":    price,          ← 제거 대상
}
```

**변경 후:**
```python
position_state[ticker] = {
```
